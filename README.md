# Bulk Notice Automation

Simple desktop app for law firms to:

- read customer data from Excel
- generate Word notices as PDF
- upload PDFs to Google Drive
- send email notices through Gmail
- send WhatsApp notifications through AiSensy
- keep logs for proof and follow-up

This guide is written for non-technical users.

## 1. What this app does

The app follows this flow:

1. Load an Excel file received from a bank or client
2. Fill a Word notice template for each row
3. Convert each notice to PDF
4. Upload each PDF to Google Drive
5. Let you review the generated notices
6. Send email and WhatsApp messages only after approval
7. Save logs for sent, failed, retried, and exported records

## 2. What you need before starting

Please keep these ready:

- Windows 10 or Windows 11
- Microsoft Word installed and activated
- Gmail account for sending emails
- Google account with 2-Step Verification enabled
- AiSensy account for WhatsApp sending
- Google Drive folder for uploaded PDFs
- Internet connection

## 3. Where to run the app from

Recommended:

1. Copy the application to a normal local folder such as `C:\NoticeAutomation\`
2. Do not run it from inside a zip file
3. Do not keep it directly in a heavily synced folder if possible

This repository already includes a packaged app:

- `dist\NoticeAutomation.exe`

You can also run from source if needed:

```powershell
py -3 -m pip install -r requirements.txt
py -3 main.py
```

### 5.4 Google Drive setup

The app uploads PDFs to Google Drive and sends the share link by WhatsApp.

Fill these fields in the app:

- `Credentials JSON`
- `Upload Folder ID`
- `Mock Mode`

How to set up Google Drive:

Phase 1: Project & API Setup
Open the Google Cloud Console.

Select an existing project or create a new one using the project dropdown menu at the top left.

Open the left navigation menu, go to APIs & Services > Library.

Search for Google Drive API, click on it, and click Enable. (Note: Make sure you are still in the correct project when you do this).

Phase 2: Service Account & Credentials
Go to APIs & Services > Credentials.

Click + CREATE CREDENTIALS at the top and select Service account.

Name your service account and click Create and Continue. (You can skip assigning project-level roles; it only needs access to the specific Drive folder, which we will handle later). Click Done.

Under the "Service Accounts" list, click on the email address of the account you just created (it will end in .iam.gserviceaccount.com).

Go to the Keys tab at the top. Click Add Key > Create new key, choose JSON, and click Create.

The JSON file will download. Move it into your app's directory.

Crucial Missing Step: Immediately add this JSON file to your .gitignore file. Never commit GCP credentials to a public or private repository.

Phase 3: Google Drive Configuration
Open your regular Google Drive and create the folder where PDFs should be uploaded.

Right-click the folder and select Share.

Paste the exact Service Account email address (the one ending in .iam.gserviceaccount.com) into the "Add people and groups" field. Set the role to Editor, uncheck "Notify people" (service accounts don't have an inbox), and click Share.

Copy the Folder ID from the browser's URL bar. (It is the long string of random letters and numbers immediately following drive/folders/).

Example folder URL:

```text
https://drive.google.com/drive/folders/1ABCxyz1234567890
```

In this example, the folder ID is:

```text
1ABCxyz1234567890
```

Important:

- Keep `Mock Mode` ON until the JSON file and folder sharing are correctly set
- If the folder is not shared with the service account email, uploads will fail

### 5.5 Advanced settings

These fields are usually fine with default values:

- `Output Folder`: where PDFs are saved
- `Log Folder`: where logs and batch records are saved
- `Restart Word every`: helps avoid Word hanging during large batches
- `Send delay min (s)` and `Send delay max (s)`: delay between sends
- `Max emails / day`: safety limit for Gmail

Recommended values:

- Restart Word every: `50`
- Send delay min: `3`
- Send delay max: `5`
- Max emails per day: `450`

## 6. Word template setup

Each client or bank can have its own Word template.

Save templates in the `templates\` folder.

Example template:

```text

To,
{{ NAME }}
Account No: {{ ACCOUNTNO }}

This is to inform you that an amount of Rs. {{ AMOUNT }} is due.

For {{ FIRM_NAME }}
{{ LAWYER_NAME }}
Date: {{ NOTICE_DATE }}
```

## 7. Create a client profile

Go to the `Profiles` tab and create one profile for each client or bank.

Steps:

1. Click `New Profile`
2. Enter a profile name
3. Select the Word template for that client
4. Enter the email subject
5. Enter the email body
6. Load a sample Excel file from that client
7. Map each required field to the correct Excel column

Usually required mappings:

- `NAME`

- `MOBILENO`

- `ACCOUNTNO`
- `AMOUNT`

## 8. Daily usage

Go to the `Workflow` tab.

Steps:

1. Select the client profile
2. Select the Excel file for the batch
3. Check the preview rows
4. Choose whether to:
   - generate PDF
   - send email
   - send WhatsApp
5. Click `Start`

The app will:

1. Read the Excel data
2. Generate PDFs
3. Upload PDFs to Drive
4. Open the preview screen

In the preview screen:

1. Open and review PDFs
2. Uncheck any row you do not want to send
3. Confirm approval
4. Click `Approve & Send`

1. Keep AiSensy mock mode ON
2. Keep Google Drive mock mode ON until Drive is configured
3. Test Gmail connection
4. Save settings

## 11. Common problems and simple fixes

### Gmail login failed

Check:

- 2-Step Verification is enabled
- you used an App Password
- you did not use your normal Gmail password

### AiSensy message not sending

Check:

- API key is correct
- template is approved in AiSensy
- template name matches exactly
- mock mode is turned OFF for real sending

### Google Drive upload failed

Check:

- JSON credentials file exists
- Drive API is enabled in Google Cloud
- upload folder ID is correct
- the Drive folder is shared with the service account email

### Word to PDF failed

Check:

- Microsoft Word is installed
- Word is activated
- all Word windows are closed before retrying

### Data is blank in the PDF

Check:

- the Excel column names are correct
- the profile mapping is correct
- the Word placeholders match the mapped field names

## 12. Best practices

- Test with 2 or 3 rows before running a full batch
- Keep a backup of `config.json` and your templates
- Use neutral WhatsApp wording for template approval
- Review PDFs before live sending
- Keep contact numbers and email IDs clean in Excel

## 13. Legal note

This app is a workflow tool. It helps generate and send notices, but it does not replace any legal service requirements that must still be done physically or through official channels.
